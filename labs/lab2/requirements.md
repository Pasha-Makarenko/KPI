## 1. Список вимог

### Функціональні вимоги (FR)

* **FR1:** Система повинна надавати повний цикл створення звіту про баг, що включає заповнення форми, прикріплення
  файлів та початкове призначення виконавця.
* **FR2:** Система повинна забезпечувати спільну роботу над багом, дозволяючи користувачам додавати коментарі та
  прикріплювати до них додаткові файли.
* **FR3:** Система повинна надавати основний робочий простір у вигляді списку багів, який можна фільтрувати для пошуку.
* **FR4:** Система повинна підтримувати повний життєвий цикл бага, дозволяючи змінювати його статус (наприклад, "In
  Progress", "Done") та призначати відповідального розробника.
* **FR5:** Детальна сторінка бага (хаб) повинна агрегувати та відображати всю пов'язану інформацію: коментарі, історію
  статусів, поточного виконавця, прикріплені файли та зв'язки з комітами.
* **FR6:** Система повинна підтримувати керування залежностями між багами, дозволяючи позначати зв'язки (дублікати,
  блокери) та відображати їх на сторінці деталей.
* **FR7:** Розробники повинні мати змогу інтегрувати роботу з кодом, прикріплюючи коміти до відповідних багів, щоб це
  відображалось на сторінці деталей.
* **FR8:** PM повинен мати інструменти управління беклогом, що включають можливість пакетного редагування багів
  безпосередньо зі списку.
* **FR9:** Інструмент пакетного редагування повинен дозволяти масову зміну статусу та масове призначення виконавця.
* **FR10:** PM повинен мати змогу генерувати звітність на основі поточного відфільтрованого списку багів.
* **FR11:** Модуль звітності повинен підтримувати експорт даних у формати PDF (для презентацій) та CSV (для аналізу).
* **FR12:** Робочий сценарій Розробника (Dev) включає: знайти баг у списку, вивчити деталі, змінити статус, додати
  коментар, прикріпити файл та зв'язати коміт.
* **FR13:** Робочий сценарій Mенеджера (PM) включає: фільтрацію списку, пакетне призначення, перегляд звітів та контроль
  статусів через сторінку деталей.
* **FR14:** Управління файлами має бути наскрізним: файли, прикріплені при створенні (`UC4`) або коментуванні (`UC3`),
  мають бути доступні для перегляду (`UC2`).

### Нефункціональні вимоги (NFR)

* **NFR1 (Завантаження сторінок):** Сторінка списку багів (`UC1`) та сторінка деталей бага (`UC2`) повинні повністю
  завантажуватись не довше ніж за 2.5 секунди.
* **NFR2 (Продуктивність фільтрації):** Застосування фільтрів (`UC10`) до списку багів повинно повертати результат не
  довше ніж за 1.5 секунди, навіть при обсязі даних до 1,000,000 записів.
* **NFR3 (Одночасні користувачі):** Система повинна стабільно підтримувати до 500 одночасно активних користувачів,
  зберігаючи час відповіді UI (NFR1, NFR2) у вказаних межах.
* **NFR4 (Безпека та доступ):** Кожен запит до API, що змінює дані, повинен вимагати валідного токена автентифікації та
  перевіряти права доступу користувача (на основі ролі PM, Dev, QA).
* **NFR5 (Доступність системи):** Система повинна мати показник доступності (Uptime) 99.9% ("три дев'ятки") протягом
  робочого часу (Пн-Пт, 9:00–19:00).
* **NFR6 (Обробка файлів):** Система повинна підтримувати завантаження файлів (`UC9`) розміром до 100 МБ кожен.
* **NFR7 (Транзакційність):** Пакетні операції (`UC11`) повинні бути транзакційними: або всі баги в пакеті успішно
  оновлюються, або (у разі помилки) жоден з них не змінюється.
* **NFR8 (Сумісність):** Веб-інтерфейс системи повинен коректно функціонувати в двох останніх мажорних версія браузерів
  Google Chrome, Mozilla Firefox та Apple Safari.

## 2. Use Cases

* **UC1:** View bug list (Перегляд списку багів)
* **UC2:** View bug details (Перегляд деталей бага)
* **UC3:** Add comment (Додавання коментаря до бага)
* **UC4:** Create bug report (Створення звіту про баг)
* **UC5:** Change bug status (Зміна статусу бага)
* **UC6:** Assign performer (Призначення виконавця)
* **UC7:** Link commit (Зв'язування коміту з багом)
* **UC8:** Generate report (Генерація звіту)
* **UC9:** Attach file (Прикріплення файлу)
* **UC10:** Filter list (Фільтрація списку багів)
* **UC11:** Batch edit bugs (Пакетне редагування багів)
* **UC12:** Link bugs (Зв'язування багів між собою)
* **UC13:** Generate PDF report (Генерація PDF звіту)
* **UC14:** Generate CSV report (Генерація CSV звіту)
